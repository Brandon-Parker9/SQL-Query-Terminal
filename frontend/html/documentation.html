System Schema

The database consists of three primary tables designed for sales and personnel tracking.

departments

Columns: dept_id (PK), dept_name, budget

employees

Columns: emp_id (PK), first_name, last_name, dept_id (FK), salary, hire_date

sales_records

Columns: sale_id (PK), emp_id (FK), sale_amount, sale_date

Security & Protection

This terminal employs a Dual-Layer Defense system:

  • Application Layer: A Python-based Regex filter intercepts queries to block destructive keywords like DROP or DELETE.
  • Database Layer: The system connects via the read_only_user, which is physically restricted from modifying data at the Postgres level.

Sample Query

-- Find top earners by department
SELECT d.dept_name, e.first_name, e.salary 
FROM employees e 
JOIN departments d ON e.dept_id = d.dept_id 
ORDER BY e.salary DESC;
